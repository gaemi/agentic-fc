# Governance

Agentic FC currently uses a maintainer-led governance model.

Maintainers are responsible for:

- setting project direction,
- reviewing and merging pull requests,
- protecting deterministic simulation and visibility rules,
- managing releases and security disclosures,
- moderating the community under the Code of Conduct.

Significant gameplay, API, persistence, or architecture changes should be
discussed publicly before implementation. The outcome should be reflected in
the relevant design document, not hidden in issue history.

This governance model may evolve as the contributor community grows.
